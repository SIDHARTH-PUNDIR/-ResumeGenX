Input File Guide

DOCS
i An .rdl file is a plain-text resume data file. Write it in any text editor — no coding knowledge needed. Follow the 5 rules below and the parser handles the rest.
Rule 1 — Assigning a value
→Use = or : to assign — both work identically.
Name = John Doe
Email : john@email.com
Rule 2 — Resume sections
→Use Section to start a group and SubSection for each entry inside it. Valid section names: Experience, Education, Projects, Skills.
Section: Experience
    SubSection = Google
Rule 3 — No quotes needed
→Type descriptions naturally. Commas, semicolons, and punctuation are treated as plain text — never as syntax.
Summary = Developer specializing in compilers, networking, and databases.
Rule 4 — Wrapping long text
→Press Enter and indent the next line with Tab or spaces — the parser joins them into one paragraph automatically.
Description = Built a fast, secure payment system.
    Reduced latency by 40% using caching.
→ Renders as one paragraph in the PDF
Rule 5 — Bullet point lists
→New line + indent + - creates a bullet. Use these for achievements and responsibilities.
Highlights:
    - Led a team of 6 engineers across 3 time zones.
    - Shipped 4 major features ahead of schedule.
Before you upload
!Avoid < > & \ " ' inside values — these break PDF rendering on export.
✓Dates must use YYYY-MM format — e.g. 2023-06.
✓Save the file as UTF-8 encoding. Max size: 5 MB.
✓File must be saved with the .rdl extension.
